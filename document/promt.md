Название проекта:
MaxNotify.

Контекст проекта:
Я разрабатываю сервис для интеграции видеонаблюдения Dahua/NVR с российским мессенджером MAX.

Тестовый стенд:
- DHI-VTO2311R-WP добавлена видеопотоком на регистратор DHI-NVR4232-4KS2/L.
- PHP-сервис будет работать с API регистратора.
- Сервис должен уметь получать snapshot с регистратора.
- Сервис должен отправлять уведомления в MAX.
- Сначала события можно имитировать через тестовый HTTP endpoint.
- Позже проект будет адаптирован под боевой объект с NVR5232-EI и Dahua-камерами с аналитикой человек/автомобиль.

Текущий стек:
- Чистый PHP
- Composer
- PSR-4 autoload
- Docker Compose
- nginx stable alpine
- php-fpm
- MySQL 8.0
- phpMyAdmin
- symfony/var-dumper
- PDO для MySQL
- Без Laravel
- Без Symfony Framework
- Без Slim, Yii, Phalcon и других фреймворков

Текущий docker-compose.yml примерно такой:

Текущий composer.json:

Главная цель проекта:
Сделать сервис, который:

1. Принимает тестовое событие через HTTP endpoint.
2. Определяет камеру, тип события, объект и время события.
3. Фильтрует события по расписанию: только с 21:00 до 06:00 по МСК.
4. Фильтрует только разрешённые камеры.
5. Делает антидубль, чтобы не спамить клиента.
6. Получает snapshot с канала Dahua NVR.
7. Отправляет текстовое сообщение в MAX.
8. Позже отправляет изображение в MAX.
9. Сохраняет событие и результат отправки в MySQL.
10. Логирует ошибки.
11. Позволяет легко заменить тестовый источник событий на реальные события от Dahua/NVR.
